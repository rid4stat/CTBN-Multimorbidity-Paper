> Olaniran, O. R., Paria, S. S., Khondoker, M., MacGregor, A., and Lewin, A.
> *Continuous-Time Bayesian Networks with Structured Shrinkage Priors for

Network (CTBN) framework.

## What is in this repository?

The repository contains:

* The full **R implementation** of the CTBN model with four order-dependent
  shrinkage priors (Hierarchical Structured / Bayesian LASSO / Regularised
  Horseshoe / Spike-and-Slab) — in two computational flavours:
    * a **Stan / NUTS MCMC** implementation (`R/ctbn_stan_fit.R`) for full
      posterior inference, and
    * a **Fast L-BFGS-B MAP + Laplace** implementation
      (`R/ctbn_map_fast.R`), 50–200× faster than MCMC, used for the
      large-scale simulation study and as a scalability proof-of-concept.
* **Simulation scripts** (`scripts/simulation/`) that calibrate to UK
  Biobank prevalence patterns and benchmark all four priors across
  interaction orders P ∈ {1, 2, 3} and penalty values θ ∈ {0.5, 1, 2}.
* **UK Biobank analysis scripts** (`scripts/ukb_analysis/`) that fit the
  recommended spike-and-slab CTBN (P = 2, θ = 1) to the InflAim cohort,
  derive posterior inclusion probabilities (PIPs), rate ratios, conditional
  risks, and synergistic excess risks.
* **All figures and tables** of the paper and supplementary, regenerated by
  the scripts (`paper/figures/`, `paper/tables/`).
* A **CONSORT diagram** of cohort derivation (`paper/figures/inflAim_consort_diagram_v2.png`).
* The **LaTeX source** of the paper and supplementary material
  (under `paper/`, copied verbatim from the OUP submission folder).

## Methodology in one minute

For *M* binary chronic conditions evolving in continuous time, the joint
process **X**(*t*) = (X<sub>1</sub>(t), …, X<sub>M</sub>(t)) is modelled as
a CTBN in which each component is a continuous-time Markov chain with an
absorbing onset state. For each target condition *m*, the log onset
intensity is

```
$log q_m(t,u) = β⁰_m  +  Σ_{ℓ ∈ Pa(m)}   β¹_{m,{ℓ}}   X_ℓ(t⁻)
                    +  Σ_{L⊆Pa(m), 2 ≤ |L| ≤ P}   β^|L|_{m,L} Π_{ℓ∈L} X_ℓ(t⁻)
                    +  Z(t)ᵀ γ_m$,
```

so that the network coefficients **β** capture main and higher-order
disease–disease effects, and **γ**<sub>m</sub> captures patient-level
covariates. Because intensities are piecewise constant between observations,
the CTBN likelihood is equivalent to an offset-Poisson regression on
at-risk intervals — the workhorse of every implementation in this
repository.

To control combinatorial growth of the interaction space, each coefficient
β<sub>j</sub> of interaction order p<sub>j</sub> receives an
**order-dependent shrinkage prior** whose effective scale contracts by
exp(−θ p<sub>j</sub>):

| Prior | Form | Selection output |
|-------|------|------------------|
| Hierarchical Structured (HS) | Normal × Inv-Gamma scale mixture | pseudo-PIP (shrinkage factor) |
| Bayesian LASSO | Laplace (DE) with order-penalised rate | pseudo-PIP (shrinkage factor) |
| Regularised Horseshoe | Global–local with slab cap (Piironen & Vehtari, 2017) | pseudo-PIP (shrinkage factor) |
| **Spike-and-Slab** *(recommended)* | Two-component normal mixture | exact PIP via Bayes' theorem |

Selection uses the PIP / pseudo-PIP at threshold 0.5; predictive performance
is assessed by Poisson log-likelihood, Brier score, and time-dependent
AUC over horizons τ ∈ {1, 2, 5, 7, 10} years.

## Installation

### System requirements

* R ≥ 4.2
* A C++17 toolchain (for Stan): Rtools (Windows), Xcode CLT (macOS), or
  build-essential (Linux)
* Recommended: ≥ 16 GB RAM for the UK Biobank-scale fits; 4 GB is enough
  for the MAP simulation study

### R packages

The dependency list is identical to the `Imports:` field in
[`DESCRIPTION`](DESCRIPTION) and can be installed in one shot:

```r
install.packages(c(
  "data.table", "rstan", "Rcpp", "RcppParallel",
  "foreach", "doParallel", "R6", "jsonlite", "crayon",
  "pROC", "timeROC", "survival",
  "ggplot2", "ggraph", "igraph", "patchwork", "scales",
  "viridisLite", "reshape2", "xtable", "stringr"
))
```

### Cloning

```bash
git clone https://github.com/InflAim/CTBN-Multimorbidity.git
cd CTBN-Multimorbidity
```

## Quick start

The smallest end-to-end example is a single MAP fit on a small simulated
dataset. From an R session at the repository root:

```r
source("R/ctbn_map_fast.R")

# Simulate a tiny dataset (uses defaults from the simulation script)
source("scripts/simulation/01_map_simulation.R")
sim_data <- generate_dataset(n = 1000, seed = 2026)
DT_wide  <- prepare_wide(sim_data)

# Fit the recommended spike-and-slab CTBN
fit <- ctbn_map_fast(
  DT_wide,
  max_order = 2L,                # P = 2
  prior     = "spike_slab",      # recommended prior
  theta     = 1.0,               # order penalty
  pi0       = 0.10               # base inclusion probability
)

summary(fit, pip_threshold = 0.5)
```

For full MCMC inference, swap `R/ctbn_map_fast.R` for `R/ctbn_stan_fit.R`
and call `ctbn_fit()` with the same arguments — see the
[Reproducing the paper](#reproducing-the-paper) section below.

## Reproducing the paper

The scripts in `scripts/` are numbered in dependency order. To regenerate
all figures and tables, run them in the following order from the
repository root:

### 1. Simulation study

```bash
Rscript scripts/simulation/01_map_simulation.R          # ~10 min on 8 cores (fast MAP)
Rscript scripts/simulation/02_map_simulation_results.R  # post-processing, plots, LaTeX
Rscript scripts/simulation/03_stan_simulation.R         # full Stan study (hours)
Rscript scripts/simulation/04_stan_simulation_results.R
```

Outputs land in `results/sim_results/` and are copied to
`paper/sim_results/` on success. Replicates are cached individually in
`results/sim_results/replicates/rep_<scenario>_<r>.rds`, so a crashed run
resumes seamlessly. See the headers of `03_stan_simulation.R` and
`01_map_simulation.R` for the global settings (`N_REP`, `K_FOLD`,
`THETA_GRID`, `P_FIT_GRID`, prior list, etc.).

### 2. UK Biobank analysis

The UK Biobank derivation pipeline assumes you have already extracted the
33,558-participant InflAim cohort following the CONSORT diagram in
`paper/figures/inflAim_consort_diagram_v2.png`. Once `DT_wide` is on disk:

```bash
Rscript scripts/ukb_analysis/04_stan_results.R   # Recommended pipeline (paper)
# or, for a fast MAP-based check:
Rscript scripts/ukb_analysis/03_map_results.R
```

## Data availability

The UK Biobank primary-care data used in this study are **not** redistributable
through this repository. Access is governed by the UK Biobank Access
Management System (UKB application 142532). Researchers wishing to
reproduce the empirical results must apply independently to UK Biobank
([https://www.ukbiobank.ac.uk/](https://www.ukbiobank.ac.uk/)). The cohort
derivation logic is reproduced in full in §4 of the paper and in the CONSORT
diagram (`paper/figures/inflAim_consort_diagram_v2.png`), and a synthetic
data generator that matches UK Biobank prevalence is provided in
`scripts/simulation/01_map_simulation.R` for methodological testing without
real-data access.

## Key results

The recommended spike-and-slab CTBN (P = 2, θ = 1) fitted to the InflAim cohort
recovers two well-separated network modules:

* a **cardiometabolic cluster** anchored by diabetes — DM → HTN
  (RR ≈ 2.13; +12.4 pp 5-year excess risk; PIP = 1.00), HTN ↔ IHD, and
  IHD → HL all with PIP ≥ 0.96;
* an **inflammatory cluster** characterised by atopic / airway-inflammatory
  pathways — CLRD → RS (RR ≈ 2.09), RS → DE (RR ≈ 1.95), RS ↔ CLRD,
  all with PIP ≥ 0.99.

All four high-PIP triplets (PIP ≥ 0.997) exhibit **sub-multiplicative
antagonism**, i.e. joint cardiometabolic risk is elevated yet attenuated
relative to the multiplicative expectation, contradicting a naïve
multiplicative-risk view of multimorbidity.

Among the four shrinkage priors, the **spike-and-slab is unambiguously
preferred**: in the simulation study it attains FPR ≈ 4 % at TPR ≈ 73 %
for interaction terms with coverage at the nominal level, whereas the
regularised horseshoe achieves selection AUC ≈ 47 % — no better than chance
— owing to indiscriminate parameter inclusion.
